# 方法论精编（白皮书数字化）

> SK0 的方法论底座，指导各子技能的提示词工程与控制策略。详见设计稿 §0 / §1 / §6。

## 一、核心理念

- **"好的提示词是导演台本，不是命令式指令"**（《丧尸清道夫》作者）。提示词应描述
  镜头意图（主体/动作/动机/构图/运镜/质感），而非罗列控制参数。
- **本地为唯一事实源**：所有文本契约（project.yaml / shotlist / genspec / takes …）
  在仓库内可重放、可审计；平台 UI 状态不可信，能力数字一律挂 `verified_at`。
- **随机性即资源**：不追求每镜 100% 锁死。叙事关键帧锁死（locked）、氛围空镜放手赌
  即兴红利（free）、多数镜头 guided。对应 `control_level` 三级（白皮书 R4 留白原则）。
- **去 AI 味第零层基座**：所有提示词强制注入真实质感正向词（超写实/胶片颗粒/真人实景）
  + 强制反向词（杜绝 CG 感/塑料皮肤/磨皮），见 `deai-keywords.yaml`。

## 二、一致性策略（取代伪参数 weight）

角色一致性主机制 = `identity_strategy`：

- `prompt_lock_zh/en`：身份锁定描述（双语，供 UI/API 两通道复用）。
- `negative_zh/en`：负向词，排除漂移方向。
- `ref_order`：参考图顺序（"顺序即权重"为社区观察，**未经官方确认**，仅作辅助）。

**不再写数值 weight**——平台并不暴露可靠的数值权重旋钮。

## 三、四层提示词结构（§6）

0. **去 AI 味层**：真实质感正向 + 反 CG 反向（deai-keywords.yaml）。
1. **设定层**：角色 feature_card（禁止徒手重写）+ 场景 + audio_cue。
2. **风格层**：从 style-bible 注入。
3. **分镜层**：景别 / 运镜 / 坐标化构图 / 动作（行为逻辑增强）。

CRAFT 八要素自查：去AI味质感 / 主体 / 动作 / 环境 / 视觉风格 / 相机 / 构图 / 情绪。

## 四、关于"标杆案例"的事实澄清（重要，设计稿 §0.1）

> 《丧尸清道夫》（刘梓瑜 / Mx-Shell）的**实际控制面**是
> "**一张场景图 + 一张角色图 + 文字指令**"——作者**不使用分镜图、不使用首尾帧**。

- "**三视图定角法**"出自社区业界总结文章，是有价值的**社区工程实践**，
  但**并非作者本人方法论**。
- 本系统据此处理：
  - "三视图 / 多视图定角"作为 SK3 的**可选**平台角色训练实践（非强制环节）。
  - "每镜必做首帧"被 `control_level` 三级分级取代（locked 做、guided 选做、free 不做）。
  - 带出处的社区启发（70/30 留白、顺序即权重、三视图等）统一标注"非官方、社区总结"。

## 五、单人吞吐与混合执行（Hybrid-Executor）

- **draft pass 默认走 API**（fal.ai / 火山方舟快档抽卡，数量级提升单人吞吐）。
- **final pass 默认走 UI**（Higgsfield Cinema Studio 终渲，导演在环、保真最高），
  除非通道经 `verified_at` 确认支持目标分辨率才可改走 API 终渲。
- 全部 API 不可用时降级为纯 UI（`fallback_on_quota_exhausted: ui_only`）。

## 六、动态编剧法

剧本/分镜未定型时允许快速试错：多提示词变体并行、频繁回流 S2/S4、用 `metrics.py`
数据校准经验。已落为可配置的 `exploration` profile（见 profiles.md）。
